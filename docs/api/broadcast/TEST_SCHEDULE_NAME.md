# 🧪 스케줄 이름 표시 테스트 가이드

## 📋 현재 상태

### 기존 파일들
- `broadcast_2025-11-20_10-11-11.wav`
- `broadcast_2025-11-20_10-25-48.mp3`
- `broadcast_2025-11-20_10-48-03.mp3`

**특징:**
- `broadcast_`로 시작하는 기존 파일
- 스케줄 이름이 없음
- 파일명 그대로 표시됨

### 새로 생성될 파일들
- `{스케줄이름}_2025-11-20_10-11-11.wav`
- 예: `8시테스트_2025-11-20_10-11-11.wav`

**특징:**
- 스케줄 이름이 파일명에 포함됨
- `list.php`에서 스케줄 이름 추출
- 플레이어에 스케줄 이름 표시

---

## 🧪 테스트 방법

### 1. 새 스케줄 생성

**웹 서비스에서:**
1. 스케줄 관리 페이지 열기
2. 새 스케줄 생성
3. 스케줄 이름 입력: "8시테스트"
4. 음원 선택 및 시간 설정
5. 저장

**기대 결과:**
- 파일명: `8시테스트_2025-11-20_10-11-11.wav`
- 플레이어에서 "8시테스트"로 표시

### 2. 플레이어에서 확인

**PC 플레이어:**
1. 플레이어 페이지 열기
2. 오디오 목록 확인
3. 새로 생성된 파일은 스케줄 이름으로 표시
4. 기존 파일은 파일명 그대로 표시

---

## ✅ 확인 사항

### 1. 파일명 형식

**새 파일:**
- ✅ `{스케줄이름}_{타임스탬프}.{확장자}`
- ✅ `list.php`에서 스케줄 이름 추출

**기존 파일:**
- ✅ `broadcast_{타임스탬프}.{확장자}`
- ✅ 파일명 그대로 표시

### 2. 표시 확인

**오디오 목록:**
- 새 파일: 스케줄 이름 표시 (큰 글씨)
- 기존 파일: 파일명 표시
- 파일명은 작은 글씨로 표시 (스케줄 이름이 있을 때만)

**현재 재생 정보:**
- 스케줄 이름이 있으면 표시
- 스케줄 이름이 없으면 파일명 표시

---

## 🔧 문제 해결

### 새 스케줄 이름이 표시되지 않는 경우

**확인 사항:**
1. `execute-schedules` 함수 재배포 확인
2. `index.php` 최신 버전 업로드 확인
3. 새 스케줄 생성 시 파일명 확인

**확인 방법:**
```
https://nanum.online/tts/api/broadcast/list.php
```

**기대 결과:**
```json
{
    "audio_list": [
        {
            "filename": "8시테스트_2025-11-20_10-11-11.wav",
            "schedule_name": "8시테스트",
            "display_name": "8시테스트",
            ...
        }
    ]
}
```
