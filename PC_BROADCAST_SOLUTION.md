# 🖥️ PC 방송 송출 솔루션

## 💡 답변

### 질문 1: PC와 RJ-45로 방송 장비 연결 시 사운드 카드 출력?

**답변: 네, 맞습니다!**

PC가 RJ-45(또는 오디오 케이블)로 방송 장비에 연결되어 있으면:
- PC의 사운드 카드 출력이 방송 장비로 전송됩니다
- PC에서 오디오를 재생하면 방송 장비로 자동 출력됩니다

### 질문 2: 엔드포인트 설정이 필요한가?

**답변: 아니요, 필요 없습니다!**

PC는 **태블릿 PC와 동일한 방식**으로 작동합니다:
- 엔드포인트 설정 불필요
- 자동 다운로드 및 재생 방식 사용

### 질문 3: 자동화로 생성된 음원을 자동 재생?

**답변: 네, 맞습니다!**

**작동 방식:**
1. 관리자가 웹에서 스케줄 설정
2. `execute-schedules` 함수가 오디오를 서버에 저장
3. PC 플레이어가 주기적으로 오디오 목록 확인
4. 새로운 오디오 발견 시 자동 다운로드 및 재생
5. PC의 사운드 카드를 통해 방송 장비로 출력

---

## 🚀 구현 방법

### PC용 플레이어 사용

**파일:** `api/broadcast/player-pc.html`

**사용법:**
1. PC에서 브라우저 열기
2. 다음 URL 접속:
   ```
   http://localhost:8000/player-pc.html
   ```
   또는
   ```
   https://tts.nanum.online/player-pc.html
   ```
3. 화면을 한 번 클릭 (자동 재생 활성화)
4. 자동 재생 시작

**기능:**
- ✅ 10초마다 오디오 목록 확인
- ✅ 새로운 오디오 발견 시 자동 재생
- ✅ PC의 사운드 카드를 통해 방송 장비로 출력
- ✅ 재생 상태 표시

---

## 📋 작동 흐름

### 1. 관리자 (웹 서비스)

```
웹 서비스 → 스케줄 생성 → 음원 선택 → 시간 설정 → 저장
```

### 2. 시스템 (자동)

```
execute-schedules 함수 → 오디오 다운로드 → 서버에 저장
```

### 3. PC (자동)

```
PC 플레이어 → 오디오 목록 확인 → 새 오디오 발견 → 다운로드 → 재생
```

### 4. 방송 장비 (자동)

```
PC 사운드 카드 → RJ-45/오디오 케이블 → 방송 장비 → 출력
```

## ✅ 태블릿 PC vs PC

### 공통점

- ✅ 동일한 플레이어 구조
- ✅ 자동 다운로드 및 재생
- ✅ 엔드포인트 설정 불필요

### 차이점

| 항목 | 태블릿 PC | PC |
|------|-----------|-----|
| **플레이어** | `player.html` | `player-pc.html` |
| **출력** | 태블릿 오디오 출력 | PC 사운드 카드 출력 |
| **연결** | RJ-45/오디오 케이블 | RJ-45/오디오 케이블 |
| **자동 실행** | 화면 고정 설정 | 시작 프로그램 설정 |

## 🔧 PC 자동 실행 설정

### Windows

**방법 1: 시작 프로그램에 추가**

1. `Win + R` → `shell:startup` 입력
2. 플레이어 바로가기 생성
3. 바로가기를 시작 프로그램 폴더에 복사

**방법 2: 작업 스케줄러 사용**

1. 작업 스케줄러 열기
2. 기본 작업 만들기
3. 트리거: 컴퓨터 시작 시
4. 작업: 프로그램 시작
5. 프로그램: 브라우저 실행
6. 인수: 플레이어 URL
